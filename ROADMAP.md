# Roadmap

FounderOS is feature-complete as of v1.22.0. This file is a stable handoff. It records what shipped, what we deliberately did not build, and where to take FounderOS if you want to extend it.

The repo no longer accepts new feature requests through issues. Critical-breakage patches may ship. Community forks are encouraged. See `CONTRIBUTING.md` for the issue-triage policy and `docs/forking.md` for extension points.

## What we deliberately did not build

Items considered, declined. Each has a reason. If you need any of these, fork.

- **Semantic embeddings on the brain layer.** Considered for v1.13. Declined because every supported embedding path required either a paid API key or a heavy local model (sentence-transformers), and the free-tier accessibility floor is a load-bearing constraint. The keyword + graph layer is sufficient for corpus sizes a single founder accumulates. If you need embeddings, write to `brain/.embeddings.jsonl` keyed on the stable IDs the v1.7 release introduced.
- **Notion Starter Kit / external cockpit.** Considered. Declined because public FounderOS stays local-first. If you want a Notion mirror, fork and add a sync script - the markdown files are designed to translate.
- **Automation packs (morning sync, client health, invoice processor).** Considered. Declined because crons proved unreliable in operator testing and shipping them publicly would damage trust. The skills exist to do the work; users invoke them on demand.
- **Continuous research cortex.** Considered. Declined because the trigger-based pattern is sufficient for solo founders; only justified at team scale.
- **Hybrid temporal knowledge graph (Neo4j + Graphiti + Letta).** Considered. Declined for the same reason as embeddings - heavy infrastructure for value that the markdown + relations.yaml layer captures at 95% fidelity for a single user.
- **Query layer caching at 500+ entries per channel.** Considered. Declined because no single founder we tested with reached that threshold. If your channel passes 500 entries, fork and add a parsed-frontmatter cache keyed on file mtime.
- **Company OS layer 2 (multi-user state model).** Considered. Declined because public FounderOS is single-user by design. Multi-user is a separate product.
- **"Works with: Cowork" tag.** Depends on Anthropic shipping hook and `.claude/commands/` parity in Claude Cowork. Out of our control; not promised.
- **`/founder-os` orchestrator skill.** Optional unified entry point considered for ergonomics. Declined to keep the command surface tight. The namespaced commands are the stable interface.
- **Visual bar formatting in status and lint.** Cosmetic. Skipped to ship sooner.
- **README banner.svg + status badges.** Cosmetic. Skipped.
- **HTML playbook.** Nice-to-have for sharing. Skipped - README plus `docs/first-day.md` cover the install case.
- **Output file naming convention doc.** Skipped - the skill bodies document their own output paths. Adding a central doc adds a drift surface.
- **Citation verification on research branch sources.** Small community task. Not worth a release of its own.

## How to fork and extend

If you want to take FounderOS further:

- **Add a skill.** Drop a `skills/<your-skill>/SKILL.md` file. The plugin loader picks it up. The setup wizard's "additional skills" prompt asks the user whether to enable it on install.
- **Add a slash command.** Drop a `.claude/commands/<your-command>.md` file. Use the existing commands as templates.
- **Add a hook.** Drop a script in `.claude/hooks/` and register it in `.claude/settings.json`. The bash + PowerShell parallel-implementation pattern from `session-start-brief.*` is the reference.
- **Add a runtime payload.** Write a Python script that emits markdown to a known path. Update the skills that should consume it (the v1.10 brain-snapshot pattern is the reference).
- **Add embeddings, Notion sync, cron automation, multi-user state.** See "What we deliberately did not build" above for the starting points.

For all extensions, preserve three rules: free-tier accessibility floor (no paid key required for core function), local-first (no mandatory external service), markdown-first (no proprietary format). See `docs/forking.md` for the full extension guide.

## Compatibility commitment

Every roadmap item shipped as an addition. Existing skills, commands, hooks, and files keep working unchanged. If a future change would alter behavior for an existing user, it goes through a deprecation cycle: announce in CLAUDE.md, ship as opt-in flag for one minor version, then promote to default in the next minor version. No silent behavior changes.
